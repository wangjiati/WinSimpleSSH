win系统下,没有SSH可用. 其他各种远程工具被杀毒软件拦截.
这里是做一个简易的SSH工具, 自定义端口(默认:22222),自定义通信协议(报文采用json传输), 使用websocket通信.  有客户端和服务端两个应用. 使用用户名+密码验证, 用户名和密码配置在服务端的配置文件中
win7, win10, win11 都需要支持.

===== 需求讨论补充 =====

【核心功能】
- 远程 Shell: 交互式 cmd.exe, 类似真实 SSH 的交互体验
- 文件传输: 支持上传(客户端→服务端)和下载(服务端→客户端)
- 不需要端口转发, 仅局域网使用
- 明文通信, 不加密

【技术选型】
- 框架: .NET Framework 4.5.2 (客户端+服务端)
- 通信: WebSocket, 报文采用 JSON 格式
- 客户端: CLI 命令行工具
- 服务端: 控制台应用

【连接与认证】
- 用户名+密码认证, 凭据配置在服务端 JSON 配置文件中
- 支持多客户端同时连接
- 10分钟无活动自动断开: 先发警告, 10秒后断开
- 检测客户端非正常断开并清理资源

【服务端运行方式】
- 双击运行(控制台模式, 有窗口), Ctrl+C 停止
- 服务端控制台命令: list(列出客户端), kick(断开客户端), help, exit

【客户端交互设计】
- 连接: ssh-cli connect <ip> -p <port> -u <username>
- 交互式 Shell, 类似 SSH 操作方式
- 支持文件传输命令: upload/download
- 支持 Ctrl+C 中断正在执行的命令
- 支持上下箭头翻历史命令
- 管理命令: clients(列出客户端), kick <id>(断开指定), kick all(断开其他所有)
- 心跳: 每30秒发送 Ping 保持连接

【文件传输】
- 支持上传和下载
- 大文件传输显示进度条

【编码处理】
- cmd.exe 默认输出 GBK 编码, 服务端转为 UTF-8 后再传输

【界面语言】
- 客户端和服务端帮助提示支持中英双语
